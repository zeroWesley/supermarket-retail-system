C端交互说明

  • 首页优先展示门店、预计送达、搜索、优惠券和高频分类，贴近即时零售/送酒类小程序的购买路径。
  • 商品卡片保留库存、活动、加购按钮，适合后续接后台配置与库存同步。
  • 下单先走模拟支付，订单支付后进入员工端待拣货，员工确认后再扣库存。
  • 酒水、生鲜、食品先在原型上体现标签，正式上线前再补年龄提示、食品/酒类资质与审核材料。

员工端交互说明

  • 工作台聚合待接单、待拣货、配送中和异常订单，适合门店员工打开后直接处理任务。
  • 订单处理支持接单/拒单，拣货页逐项核验，完成后再人工确认扣减库存。
  • 扫码出入库用于手机扫描商品条码，第一版不接扫码枪、电子秤、小票机等线下硬件。
  • 配送任务按距离和承诺送达时间排序，员工自配送后手动确认送达。

管理端交互说明

  • 管理端只处理系统账号、角色、菜单权限和操作日志，不承载日常运营配置。
  • 账号支持新增、删除、停用、重置密码、绑定门店和分配角色。
  • 菜单权限按角色配置，同时保留数据范围：全部门店、指定门店、本门店。
  • 操作日志记录账号、权限、库存、活动、订单等关键动作，方便线下演示和后续审计。

运营端交互说明

  • 运营端作为业务数据源头，商品、库存、营销活动配置后同步到 C端和员工端。
  • 库存允许人工确认，演示版重点展示可售库存、预警、出入库记录和人工修正。
  • 财务先弱化为订单金额与模拟退款字段，后续再补微信支付、退款、对账。
  • 多门店按单个超市自营处理，保留门店维度，暂不做多商户 SaaS。
